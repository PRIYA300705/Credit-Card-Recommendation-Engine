# AI-Powered Credit Card Recommendation Engine

An end-to-end product case study that reimagines how users discover and choose credit cards. Instead of relying on lengthy forms and generic eligibility checks, the experience uses spending behaviour, credit signals, and user preferences to deliver personalised recommendations in under 90 seconds.

This project covers the complete product lifecycle, from problem discovery and product strategy to user stories, prioritisation, recommendation logic, and an interactive prototype.

## Live Prototype

https://credit-card-recommendation.netlify.app/

## What’s Included

### Product Requirements Document (PRD)

* Defined the problem statement, product goals, success metrics, and user personas.
* Designed a behaviour-driven recommendation framework based on spending patterns, eligibility, and reward optimisation.
* Mapped the complete user journey from onboarding and consent collection to recommendation and application handoff.
* Documented the technical architecture, recommendation engine, explainability layer, and data strategy.

### User Stories & Prioritisation

* Created 22 user stories across onboarding, financial analysis, recommendation, explainability, comparison, and application workflows.
* Defined detailed acceptance criteria for each story.
* Prioritised features using the MoSCoW framework to establish a clear MVP scope.

### Interactive Prototype

A mobile-first prototype demonstrating:

* Smart onboarding and consent collection
* Financial profile analysis
* Personalised card recommendations
* Recommendation explanations
* Eligibility confidence indicators
* Card comparison experience
* Application handoff flow

## Key Product Decisions

### Minimise User Effort

Rather than asking users to fill long questionnaires, the system uses available financial signals and only asks a few contextual questions when required.

### Explain Every Recommendation

Each recommendation is accompanied by clear reasons that help users understand why a card was suggested, improving transparency and trust.

### Design for Trust

Users are informed about how their data is used, what permissions are required, and how recommendations are generated.

### Graceful Fallbacks

If users choose not to share certain data sources, the experience adapts and still provides useful recommendations through a lightweight preference flow.
